* Motivation

Provider and model management currently has no in-app interface — users must
restart with CLI flags. A slash command system (like Claude Code's ~/model~,
~/clear~, etc.) lets users manage accounts, switch models, and control the
session without leaving the input prompt.

* Scope

When the user types ~/~ at the start of the input buffer, kodo enters slash
command mode: an autocomplete list appears above the input, narrows as the user
types, and the selected command executes on Enter. Escape or backspace-to-empty
cancels back to normal input.

Built-in commands:

| Command                    | Effect                                      |
|----------------------------+---------------------------------------------|
| ~/help~                    | Print all available commands                |
| ~/clear~                   | Clear conversation history                  |
| ~/compact~                 | Summarise context and replace messages      |
| ~/model [id]~              | Show current model, or switch to ~id~       |
| ~/providers~               | List all connected accounts                 |
| ~/login <provider> [name]~ | Add a new account (OAuth or API key prompt) |
| ~/logout <account_id>~     | Remove an account from keychain + DB        |

* Current State

- Input is a plain text buffer; no special handling of leading ~/~.
- ~command.rs~ has a ~PaletteCommand~ list used for the ~Ctrl+P~ palette.
- No slash command parser, no inline autocomplete.

** TODO Step 4: Slash command parser

*What*: Parse a raw input string (after the leading ~/~) into a ~ParsedSlash~
with a command name and argument tokens.

*File*: ~crates/kodo-ui/src/slash.rs~ (append)

- ~ParsedSlash~ struct: ~name: String~, ~args: Vec<String>~
- ~parse(input: &str) -> ParsedSlash~ — splits on whitespace; first token is
  name (without ~/~), rest are args
- ~is_slash_input(input: &str) -> bool~ — true iff input starts with ~/~ and
  cursor is at beginning of input (no preceding text)

** TODO Step 5: Slash UI state

Add to ~Model~:

- ~SlashState~: ~completions: Vec<&'static SlashCommand>~, ~selected: usize~

Activation logic (in ~update.rs~, on each keystroke):
- If ~is_slash_input(input)~: set ~slash_state~ with filtered completions.
- If input no longer starts with ~/~: clear ~slash_state~.

** TODO Step 7: Update handler — navigation and dispatch

*What*: Handle slash ~Msg~ variants in ~update.rs~ and route to command logic.

*File*: ~crates/kodo-ui/src/update.rs~

- ~SlashNav~: adjust ~slash_state.selected~ with wrap.
- ~SlashCancel~: clear ~slash_state~; leave input unchanged.
- ~SlashExecute~:
  1. Parse the current input with ~slash::parse()~.
  2. Clear input and ~slash_state~.
  3. Match on ~name~:
     - ~"help"~ → append help text to messages
     - ~"clear"~ → clear ~model.messages~
     - ~"compact"~ → dispatch ~AppCmd::CompactContext~
     - ~"model"~ with arg → dispatch ~AppCmd::SetModel(id)~; without arg →
       append current model info to messages
     - ~"providers"~ → dispatch ~AppCmd::ListProviders~; result arrives as
       ~Msg::ProvidersListed(accounts)~ → append formatted list to messages
     - ~"login"~ → dispatch ~AppCmd::LoginProvider { provider, name }~; result
       arrives as ~Msg::LoginComplete(account)~ → append confirmation message
     - ~"logout"~ → dispatch ~AppCmd::LogoutProvider(account_id)~; result
       arrives as ~Msg::LogoutComplete(account_id)~ → append confirmation
     - unknown command → append error message

Also intercept Enter key in ~update.rs~: if ~slash_state.is_some()~, dispatch
~SlashExecute~ instead of sending the input as a chat message.
