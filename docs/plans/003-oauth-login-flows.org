* Motivation

Auth currently requires manually setting environment variables
(~ANTHROPIC_API_KEY~, ~OPENAI_API_KEY~, etc.). This is friction for new users
and doesn't support token refresh. This plan removes the env var requirement
entirely — credentials are stored in the OS keychain and obtained via the
~--login~ flow.

* Scope

Implement per-provider auth strategies:
- *OpenAI*: OAuth2 Authorization Code + PKCE (matching the flow Codex CLI uses).
- *Anthropic*: API key only (keychain -> prompt via ~--login~). Anthropic does
  not expose a public OAuth flow for direct API access.
- *Gemini*: API key via ~--login gemini~ prompt, stored in keychain.

A unified ~resolve_auth()~ chain handles: keychain (with refresh if supported)
-> error with instructions to run ~--login~.

* Provider Auth Matrix

| Provider  | Auth Method        | OAuth Support | Token Refresh          | Header Format                   |
|-----------+--------------------+---------------+------------------------+---------------------------------|
| OpenAI    | OAuth (PKCE)       | Yes           | Yes                    | ~Authorization: Bearer <token>~ |
| Anthropic | API key (keychain) | No            | No (keys don't expire) | ~x-api-key: <key>~              |
| Gemini    | API key (keychain) | No            | No                     | ~x-goog-api-key: <key>~         |

* Current State

- Providers use ~from_env()~ to read env vars (anthropic.rs:237, openai, gemini).
- ~kodo-store/auth.rs~ has ~AuthToken~ struct with ~token~, ~refresh_token~,
  ~expires_at~. Secrets go to OS keychain via ~keyring~ crate.
- ~auth_providers~ table in SQLite stores only provider name + expiry.
- No OAuth client, no browser flow, no token refresh.

* Reference: Codex CLI OAuth Flow

The Codex CLI (https://github.com/openai/codex) implements OAuth2 Authorization
Code + PKCE against OpenAI's auth infrastructure. We replicate this flow for
our OpenAI provider.

** Key Constants

- *Issuer*: ~https://auth.openai.com~
- *Client ID*: ~app_EMoamEEZ73f0CkXaXp7hrann~ (public client, no secret)
- *Authorize URL*: ~https://auth.openai.com/oauth/authorize~
- *Token URL*: ~https://auth.openai.com/oauth/token~
- *Scopes*: ~openid profile email offline_access~
- *PKCE*: S256, 64-byte random verifier, base64url-encoded (no padding)

** Authorization Code Flow (primary)

1. Start local HTTP server on a random port.
2. Build authorize URL with ~response_type=code~, ~client_id~, ~redirect_uri~,
   ~scope~, ~code_challenge~ (S256), ~code_challenge_method=S256~, ~state~.
3. Open user's browser to authorize URL.
4. Receive callback at ~/auth/callback~ with ~code~ and ~state~ params.
5. Exchange code for tokens via POST to token URL:
   #+begin_src text
   grant_type=authorization_code
   code={authorization_code}
   redirect_uri={redirect_uri}
   client_id={client_id}
   code_verifier={pkce_verifier}
   #+end_src
6. Response contains: ~id_token~, ~access_token~, ~refresh_token~.

** API Key Exchange (post-login)

After obtaining tokens, Codex performs a second token exchange (RFC 8693) to
get an actual OpenAI API key:

#+begin_src text
POST https://auth.openai.com/oauth/token
grant_type=urn:ietf:params:oauth:grant-type:token-exchange
client_id={client_id}
requested_token=openai-api-key
subject_token={id_token}
subject_token_type=urn:ietf:params:oauth:token-type:id_token
#+end_src

Response: ~{ access_token }~ — this is the API key used for subsequent requests.

** Token Refresh

#+begin_src text
POST https://auth.openai.com/oauth/token
Content-Type: application/json
{
  "client_id": "app_EMoamEEZ73f0CkXaXp7hrann",
  "grant_type": "refresh_token",
  "refresh_token": "{refresh_token}"
}
#+end_src

Codex proactively refreshes when the access token is within 8 minutes of expiry.

** Device Code Flow (headless fallback)

For SSH/headless environments where a browser can't be opened locally:

1. POST to ~{issuer}/api/accounts/deviceauth/usercode~ with ~{ client_id }~
   → returns ~{ device_auth_id, user_code, interval }~.
2. Display: "Go to https://auth.openai.com/codex/device and enter code: {user_code}".
3. Poll ~{issuer}/api/accounts/deviceauth/token~ with ~{ device_auth_id, user_code }~
   every ~interval~ seconds (15 min timeout).
4. On success: returns ~{ authorization_code, code_challenge, code_verifier }~.
5. Exchange code using the same token endpoint.

* Reference: Gemini API Auth

- HTTP header: ~x-goog-api-key: {key}~.
- Obtain at https://aistudio.google.com/app/apikey.
- No expiry, no refresh needed.
- Sufficient for all generation endpoints.

** DONE Step 4: Add ~refresh_if_needed()~ to ~auth.rs~

*What*: Check token expiry and refresh if needed. Only applies to OAuth tokens
(OpenAI); API keys (Anthropic, Gemini) never expire.

*File*: ~crates/kodo-store/src/auth.rs~

~refresh_if_needed(pool, store, account_id, token_url, client_id)~:
1. Load account from keychain; return ~None~ if not found.
2. If no ~expires_at~: return token as-is (API key, never expires).
3. If expires more than 8 minutes from now: return token as-is.
4. If expired and no ~refresh_token~: return ~None~.
5. POST refresh to ~token_url~:
   - OpenAI: JSON body ~{ client_id, grant_type: "refresh_token", refresh_token }~
   - Standard: form-encoded same fields
6. On success: if OpenAI and response has ~id_token~, re-exchange for API key.
7. Compute new ~expires_at~ from ~expires_in~; call ~update_token()~; return new token.
