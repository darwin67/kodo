* Motivation

Currently Kodo runs a single agent with one conversation context. Complex tasks
benefit from spawning isolated "subagents" — separate agent instances with their
own context, model, and tool permissions. For example, a main agent could spawn
a research subagent to explore a codebase while the main context stays focused.

* Scope

Add a ~spawn_agent~ tool that the LLM can call to start an isolated agent task.
The subagent runs to completion and returns its result as a tool output.

* Current State

- ~Agent~ (agent.rs:81-92) is a self-contained struct with its own provider,
  messages, context tracker, tools, etc.
- ~Agent::process_message()~ (agent.rs:164-202) runs the full agentic loop.
- ~ToolRegistry~ (registry.rs) stores tools in a ~HashMap<String, Arc<dyn Tool>>~.
- ~Tool~ trait (tool.rs:33-52) requires ~Send + Sync~, returns a pinned future.
- ~ToolContext~ (tool.rs:25-28) has only ~working_dir: PathBuf~.

#+begin_src rust
use std::sync::Arc;

use anyhow::{Result, bail};

use crate::tool::{PermissionLevel, Tool, ToolContext, ToolOutput};

/// Tool that spawns an isolated agent to handle a subtask.
/// The subagent gets its own conversation context and runs to completion.
pub struct SpawnAgentTool;

impl Tool for SpawnAgentTool {
    fn name(&self) -> &str {
        "spawn_agent"
    }

    fn description(&self) -> &str {
        "Spawn an isolated agent to handle a subtask. The subagent runs independently \
         with its own conversation context and returns the result. Use this for \
         research tasks, code exploration, or any subtask that benefits from \
         a separate context window."
    }

    fn parameters_schema(&self) -> serde_json::Value {
        serde_json::json!({
            "type": "object",
            "properties": {
                "task": {
                    "type": "string",
                    "description": "The task description for the subagent to accomplish"
                },
                "model": {
                    "type": "string",
                    "description": "Optional model override for the subagent (defaults to parent's model)"
                }
            },
            "required": ["task"]
        })
    }

    fn permission_level(&self) -> PermissionLevel {
        PermissionLevel::Execute
    }

    fn execute(
        &self,
        params: serde_json::Value,
        ctx: &ToolContext,
    ) -> std::pin::Pin<Box<dyn std::future::Future<Output = Result<ToolOutput>> + Send + '_>> {
        let task = params.get("task").and_then(|v| v.as_str()).map(String::from);
        let model = params.get("model").and_then(|v| v.as_str()).map(String::from);
        let working_dir = ctx.working_dir.clone();
        let provider = ctx.provider.clone();

        Box::pin(async move {
            let task = match task {
                Some(t) => t,
                None => bail!("missing required parameter: task"),
            };

            let provider_any = match provider {
                Some(p) => p,
                None => bail!("spawn_agent requires a provider in the tool context"),
            };

            // Downcast to Provider trait object
            // The provider is stored as Arc<dyn Any> to avoid circular deps.
            // We need to get it back to Arc<dyn Provider>.
            // This is done by the agent passing it correctly.
            use kodo_llm::provider::Provider;
            let provider: Arc<dyn Provider> = match provider_any.downcast::<Box<dyn Provider>>() {
                Ok(p) => {
                    // This path won't work directly — see alternative approach below
                    bail!("provider downcast not supported in this configuration")
                }
                Err(_) => bail!("failed to downcast provider — spawn_agent unavailable"),
            };

            // NOTE: The downcast approach above is complex. A simpler alternative
            // is implemented in Step 3 below using a callback pattern.
            unreachable!()
        })
    }
}
#+end_src

*Note*: The ~dyn Any~ downcast is tricky with trait objects. Step 3 replaces
this with a cleaner callback approach.

let provider = Arc::clone(&self.provider);
let model = self.model.clone();

let spawn_fn: SpawnAgentFn = Arc::new(move |task, model_override| {
    let provider = Arc::clone(&provider);
    let agent_model = model_override.unwrap_or_else(|| model.clone());
    Box::pin(async move {
        let mut subagent = Agent::new(provider).with_model(agent_model);
        kodo_tools::register_builtin_tools(subagent.tool_registry_mut());
        subagent.process_message(&task, None).await?;
        // Return the last assistant message as the result
        let result = subagent.messages
            .iter()
            .rev()
            .find(|m| m.role == Role::Assistant)
            .map(|m| m.text())
            .unwrap_or_default();
        Ok(result)
    })
});

fn execute(
    &self,
    params: serde_json::Value,
    ctx: &ToolContext,
) -> std::pin::Pin<Box<dyn std::future::Future<Output = Result<ToolOutput>> + Send + '_>> {
    let task = params.get("task").and_then(|v| v.as_str()).map(String::from);
    let model = params.get("model").and_then(|v| v.as_str()).map(String::from);
    let spawn_fn = ctx.spawn_agent.clone();

    Box::pin(async move {
        let task = match task {
            Some(t) => t,
            None => bail!("missing required parameter: task"),
        };

        let spawn = match spawn_fn {
            Some(f) => f,
            None => bail!("subagent spawning is not available in this context"),
        };

        let result = spawn(task, model).await?;

        Ok(ToolOutput {
            content: result,
            success: true,
        })
    })
}
