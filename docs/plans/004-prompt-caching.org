* Motivation

Without prompt caching, every Anthropic API request re-processes the full
system prompt + tool definitions + entire message history. Anthropic's prompt
caching lets us mark stable prefixes as cacheable, reducing cost and latency.

* Scope

Add ~cache_control~ annotations to the Anthropic provider's request types.
This is entirely Anthropic-specific — no changes to the provider-agnostic types.
Also extend ~Usage~ to track cache metrics.

* Current State

- ~ApiRequest~ (anthropic.rs:23-33) has no ~cache_control~ fields.
- ~ApiContentBlock~ (anthropic.rs:41-58) has no ~cache_control~ field.
- ~ApiUsage~ (anthropic.rs:74-80) only has ~input_tokens~ and ~output_tokens~.
- The system prompt is sent as ~Option<String>~ in ~ApiRequest~ (line 28).
